# Sections explained

**`/data`**
Mock persistence in place of actual DB connections. Like JSON.

**`/NOTES`**
Random notes and brain dumps.

**`src/domain`**
What things are. Just types/structures/models.

**`src/services`**
What things do. Business logic, classes, functions etc.

**`src/storage`**
How data is persisted. FS first, but prob connect to a postgres container down the line.

**`src/experiments`**
Scratchpad and generally chaos. Trying TS features, messing with random ideas, not very well structured.

**`src/main.ts`**
App's entry point/runner
